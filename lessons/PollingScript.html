PollingScript

PollingScript is the most basic type of script in RSBot. It offers a method, poll(), that loops until the script is stopped. The main logic of your script will be placed inside the poll() method.
In addition to the poll() method, PollingScript offers the methods start() and stop().
The start() method is executed only once when the script is started. This is the method that should be used to set up your script (setting start XP, launching GUIs, etc).
The stop() method is executed only once when the script is stopped.

Script Manifest

The Script Manifest is where details of the script, like its name and description, are stored. For a script to appear locally or on the SDN, it must have a manifest. There are 3 basic parts to the manifest: the name, the description, and the properties.

The name flag is the name of the script. This is what the script is listed as on the SDN and the script selector. The description flag is the description of the script. It shows below the name on the SDN, and below the script selector in the client.

ClientContext

ClientContext will be discussed in the next chapter, but it needs to be touched on briefly here.
ClientContext is the class that stores all of the API classes that interface with the game. For the PollingScript parameterization, it signifies which type of context your script will be using, and is part of the identification process for the SDN to determine if your script is for OS or RS3.
