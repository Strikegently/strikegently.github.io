Npcs and GameObjects

Npcs and GameObjects are used almost entirely the same, minus a few specific methods. They are both accessed from their respective query class (Npcs and GameObjects) which both extend MobileIdQuery, so their query usage is almost exactly the same as well. To reduce redundancy, they were merged into one page.

Npcs

Npcs are in-game representations of Non-Player Characters. All Npcs in the game can be accessed from the Npc query class (ctx.npcs). This class has all of the same methods as a standard query (see Query Introduction), and is used in the same manner.

GameObjects

GameObjects are an in-game representation of objects within the game, like bank booths (but not bankers, since they are Npcs), chests, and doors. Like Npcs, the GameObject query class (ctx.objects) extends MobileIdQuery, so it has the same methods.

IDs

Npcs and GameObjects have a unique number that can be used to identify them. The vast majority of the time, you will use IDs to find an Npc or GameObject. You can find an Npc's ID by going to View > Npcs, and a GameObject's by going to View > Objects in the client. Their ID (as well as other relative information) will be displayed over them. You can use the id(int...) method in the query to filter based on ID.

Name

MobileIdQuery inherits from Nameable, which provides a name to the entities via the name() method. You can use the name(String...) method in the query to filter based on name.

Location

MobileIdQuery also inherits the tile() method from Locatable, and the at(Tile) and within(double) methods from Locatable.Query. You can use the at(Tile) method to filter the query results to filter results based on the target's specific location, and the within(double) to filter results based on distance from the local player to the target.
