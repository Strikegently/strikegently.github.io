Widgets and Components

Widgets and Components are the in-game buttons and text displays. For instance, all of the tabs in the Hud (Skills, Gear, Inventory, etc.) are all made up of Components. As a general rule of thumb, a Component is contained within (or acts as as child to) a Widget.

Finding and Getting Components

In the client, you can go to View > Widgets and the Widget Viewer tool will open. This tool allows you to browse the currently valid widgets and components. When you select a widget, it will be outlined with a yellow rectangle. As you go further down the tree (Widget > Component > Component > ...), the identifying rectangle to update to the reflect the currently selected component. Each component has a unique index value that is used to identify it. You will use this index to fetch it for use. Once you have the indices of the component you're looking for, you can use it with the widget API to fetch its instance.

To fetch a specific Widget instance, you can use the widget(int index) method:
Widget widget = ctx.widgets.widget(0);

To fetch a specific Component, you use the component(int widget, int component) method:
Component component = ctx.widgets.component(0, 1);

For a nested Component (component within a component), you can use the component(int component) method in the Component class:
Component component = ctx.widgets.component(0, 1).component(2);

Chat API

The API includes a set of helper functions specifically built for traversing the chat widgets. It's used similarly to any other query. The text(String...) method will filter currently available chat options by text. The poll() method will return a ChatOption instance, which you can then select(boolean) to select. Passing true will press the key associated with the option to activate it. Alternatively, passing false will use the mouse to click the option.
ChatOption chat = ctx.chat.select().text(String...)poll();
chat.select(true);
